  <action>
    Create the public catalog page as a pure Astro component with no React island. All data fetching is server-side in the frontmatter.

    Key design constraints from CONTEXT.md (all locked):
    - Layout fills viewport height below header — use a flex column container with height: calc(100vh - 4rem) as initial estimate, iframe gets flex: 1
    - Slim header bar above iframe: "Product Catalog" heading on the left, download button on the right — always visible, never conditional
    - Empty state when no active catalog: friendly message, no 404
    - Download button: use <a href={pdfUrl} download> — the download attribute prompts save-to-disk on supporting browsers; serves as mobile fallback since iframe may not render inline

    Pitfall from RESEARCH.md: UploadThing CDN X-Frame-Options is unconfirmed. The download button being always-visible mitigates this — if the iframe renders blank, the user sees the download button immediately.

    ```astro
    ---
    import { db } from "@/lib/db";
    import { productCatalogs } from "@/lib/db/schema";
    import Layout from "@/layouts/default.astro";
    import { eq } from "drizzle-orm";

    const [activeCatalog] = await db
      .select()
      .from(productCatalogs)
      .where(eq(productCatalogs.isActive, true))
      .limit(1);
    ---

    <Layout title="Product Catalog" description="View and download our current product catalog">
      {activeCatalog ? (
        <div class="catalog-wrapper">
          <div class="catalog-header">
            <h1 class="catalog-title">Product Catalog</h1>
            <a
              href={activeCatalog.pdfUrl}
              download
              class="catalog-download"
            >
              <svg xmlns="http://www.w3.org/2000/svg" width="16" height="16" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" aria-hidden="true">
                <path d="M21 15v4a2 2 0 0 1-2 2H5a2 2 0 0 1-2-2v-4"/>
                <polyline points="7 10 12 15 17 10"/>
                <line x1="12" x2="12" y1="15" y2="3"/>
              </svg>
              Download PDF
            </a>
          </div>
    ```

    Implementation notes:
    - The inline SVG icons avoid importing lucide-react into an Astro page (Astro server components don't process React SVG components without a framework directive)
    - Use scoped `<style>` — Astro automatically scopes these to the component
    - The `download` attribute on the anchor triggers browser save-to-disk behavior where supported; it also serves as the primary mobile fallback since many mobile browsers cannot render PDFs in iframes
    - The catalog-wrapper height uses calc(100vh - 4rem) as the initial estimate per RESEARCH.md; this assumes the Header is approximately 4rem tall. If the Header is taller, increase this value.
