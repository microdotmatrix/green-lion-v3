**New render structure — replace the `{isLoadingProduct && isEditing ? ... : <form>...</form>}` block:**

```tsx
{isLoadingProduct && isEditing ? (
  <div className="space-y-4 py-4">
    <Skeleton className="h-10 w-full" />
    <Skeleton className="h-10 w-full" />
    <Skeleton className="h-20 w-full" />
  </div>
) : isEditing ? (
  <Tabs defaultValue="basic">
    <TabsList className="w-full">
      <TabsTrigger value="basic" className="flex-1">Basic Info</TabsTrigger>
      <TabsTrigger value="attributes" className="flex-1">Attributes</TabsTrigger>
    </TabsList>
    <TabsContent value="basic">
      <form onSubmit={handleSubmit} className="space-y-4">
        {error && (
          <div className="text-sm text-destructive bg-destructive/10 p-3 rounded-md">
            {error instanceof Error ? error.message : "Something went wrong"}
          </div>
        )}
        {/* ALL existing form fields — sku, category, name, description, ImageUpload,
            moq, increment, logoCost, packagingCost, pricingTiers — UNCHANGED */}
        <ResponsiveModalFooter>
          <Button type="button" variant="outline" onClick={() => onOpenChange(false)}>
            Cancel
          </Button>
          <Button type="submit" disabled={isLoading}>
            {isLoading ? "Saving..." : "Update"}
          </Button>
        </ResponsiveModalFooter>
      </form>
    </TabsContent>
    <TabsContent value="attributes">
      <ProductAttributesTab
        productId={productId!}
        categoryId={formData.categoryId}
      />
    </TabsContent>
  </Tabs>
) : (
  <form onSubmit={handleSubmit} className="space-y-4">
    {error && (
      <div className="text-sm text-destructive bg-destructive/10 p-3 rounded-md">
        {error instanceof Error ? error.message : "Something went wrong"}
      </div>
    )}
    {/* ALL existing form fields — UNCHANGED */}
    <ResponsiveModalFooter>
      <Button type="button" variant="outline" onClick={() => onOpenChange(false)}>
        Cancel
      </Button>
      <Button type="submit" disabled={isLoading}>
        {isLoading ? "Saving..." : "Create"}
      </Button>
    </ResponsiveModalFooter>
  </form>
)}
```

Key constraints:
- The `<form onSubmit={handleSubmit}>` tag is INSIDE `TabsContent value="basic"` — not wrapping the entire Tabs component. Attribute mutations are independent, not form submit.
- `productId!` is safe inside the `isEditing ? (...)` branch because `isEditing = !!productId`.
- `formData.categoryId` is the live local state value — it updates when the admin changes the category Select, giving the Attributes tab a preview of what the new category contributes (per research pitfall 2).
- The `ResponsiveModalContent` className stays `"sm:max-w-[600px]"` — no change needed.
- The `onInteractOutside={(event) => event.preventDefault()}` on ResponsiveModalContent stays — no change.
