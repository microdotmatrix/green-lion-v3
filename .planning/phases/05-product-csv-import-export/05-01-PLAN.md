```typescript
const csvRowSchema = z.object({
  sku: z.string().min(1, "sku is required"),
  name: z.string().min(1, "name is required"),
  description: z.string().optional().default(""),
  category: z.string().optional(),
  min_order_qty: z.coerce.number().int().positive().optional().default(1),
  order_qty_increment: z.coerce.number().int().positive().optional().default(1),
  logo_cost: z.string().optional().default("0"),
  packaging_cost: z.string().optional().default("0"),
  image_url: z.string().url().optional().or(z.literal("")),
  tier1_min_qty: z.coerce.number().int().positive().optional(),
  tier1_price: z.string().optional(),
  tier2_min_qty: z.coerce.number().int().positive().optional(),
  tier2_price: z.string().optional(),
  tier3_min_qty: z.coerce.number().int().positive().optional(),
  tier3_price: z.string().optional(),
  tier4_min_qty: z.coerce.number().int().positive().optional(),
  tier4_price: z.string().optional(),
  tier5_min_qty: z.coerce.number().int().positive().optional(),
  tier5_price: z.string().optional(),
});
export type CsvRow = z.infer<typeof csvRowSchema>;
```

**3. `buildExportRows` helper** — takes the joined product+category+tiers query result and returns flat objects keyed by CSV_COLUMNS names, ready for `Papa.unparse()`. Logic:
- `category`: category name string or `""` if null
- `image_url`: `images[0] ?? ""`
- `tier1_min_qty` through `tier5_min_qty`/`tier5_price`: pull from sorted tiers array (sort by `minQuantity` asc), fill up to 5, leave blank string for missing slots
- Normalize price values: strip anything that isn't `[0-9.-]` before storing

**Price normalization helper** (inline or exported):
```typescript
export function normalizePrice(raw: string | undefined): string {
  if (!raw) return "0";
  const cleaned = raw.replace(/[^0-9.-]/g, "");
  const num = parseFloat(cleaned);
  return isFinite(num) ? num.toFixed(2) : "0";
}
```
This prevents `$1.50` or `1,500.00` from breaking decimal DB columns.

**File 1: `src/pages/api/admin/products/import.ts`**

POST endpoint accepting `multipart/form-data` with a `file` field.

Full implementation logic:

```
1. Auth guard: if (!locals.user || !locals.session) → 401
2. formData = await request.formData()
3. file = formData.get("file") as File; if !file → 400
4. rawText = await file.text()
5. csvText = rawText.replace(/^\uFEFF/, "")  // strip Excel BOM (critical — see RESEARCH.md Pitfall 1)
6. parsed = Papa.parse<Record<string, string>>(csvText, { header: true, skipEmptyLines: true, transform: (v) => v.trim() })
7. Pre-fetch all categories once into Map<string (lowercase), string (id)>:
     const existingCats = await db.select({ id, name }).from(categories)
     const categoryMap = new Map(existingCats.map(c => [c.name.toLowerCase(), c.id]))
8. Process rows (index 0-based, report as 1-based + 1 for header):
   For each row at index i:
     a. rowNum = i + 2  (1=header, 2=first data row)
     b. validated = csvRowSchema.safeParse(row)
     c. If !validated.success → push to skipped with rowNum, row.sku, and first error message; continue
     d. Resolve categoryId:
        - If validated.data.category is non-empty string:
          - key = validated.data.category.toLowerCase()
          - If categoryMap.has(key) → use existing id
          - Else: insert new category (name only), get id, add to categoryMap
        - Else: categoryId = null
     e. Build product insert object:
        { sku, name, description, categoryId, minimumOrderQuantity, orderQuantityIncrement, logoCost: normalizePrice(logo_cost), packagingCost: normalizePrice(packaging_cost), images: image_url ? [image_url] : [] }
     f. Collect valid products into validRows array (with their tier data)

9. Batch upsert all valid products using onConflictDoUpdate targeting products.sku:
     const upserted = await db.insert(products)
       .values(validRows.map(r => r.product))
       .onConflictDoUpdate({
         target: products.sku,
         set: {
           name: sql`excluded.name`,
           description: sql`excluded.description`,
           categoryId: sql`excluded.category_id`,
           minimumOrderQuantity: sql`excluded.minimum_order_quantity`,
           orderQuantityIncrement: sql`excluded.order_quantity_increment`,
           logoCost: sql`excluded.logo_cost`,
           packagingCost: sql`excluded.packaging_cost`,
           updatedAt: new Date(),
         },
       })
       .returning({ id: products.id, sku: products.sku })

   Track inserted vs updated: before upsert, do a single SELECT for all skus in validRows to get a Set of existing skus. Rows whose sku is in the set → updated; others → inserted.

10. For each upserted product, replace pricing tiers (tier replace strategy):
    - Extract tiers from that row's CSV data (tier1_min_qty/tier1_price through tier5): skip pairs where min_qty or price is absent/zero
    - await db.delete(pricingTiers).where(eq(pricingTiers.productId, productId))
    - If tiers.length > 0: await db.insert(pricingTiers).values(tiers)

11. Return JSON:
    { inserted: insertedSkus.length, updated: updatedSkus.length, skipped: [{ row, sku, reason }] }
```
